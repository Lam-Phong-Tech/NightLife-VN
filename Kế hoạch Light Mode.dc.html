1. Phạm vi & chiến lược

Phạm vi: toàn bộ giao diện người dùng public ở cả 2 breakpoint — trang chủ, tìm quán (danh sách + chi tiết quán), cast (danh sách + hồ sơ), bảng xếp hạng, ưu đãi, blog — cùng chrome dùng chung: header sticky, footer, bottom nav mobile, drawer thông báo, toast, chat hỗ trợ, language picker, mobile simulator.

Chiến lược 4 phase, ship an toàn bằng cờ môi trường:

  1. P0 — Nền tảng: token 2 theme vào globals.css, script chống chớp vào layout.tsx, gỡ engine theme.js cũ, thêm component ThemeToggle (ẩn sau cờ NEXT_PUBLIC_LIGHT_THEME).
  2. P1 — Chrome chung + UI kit: SiteChrome và 8 component ui/ chuyển hết màu hardcode → token.
  3. P2 — Từng trang theo thứ tự lưu lượng: trang chủ → danh sách quán → chi tiết quán → danh sách cast → hồ sơ cast → xếp hạng → ưu đãi → blog.
  4. P3 — Dọn globals.css (các block hardcode desktop + mobile) + QA 2 theme × 2 breakpoint → bật cờ, xóa guard.

Cách chuyển đổi cơ học và ít rủi ro: codebase dùng 2 pattern màu — (a) object const colors = {...} đầu file rồi tham chiếu trong inline style, (b) chuỗi CSS lớn render qua <style>{cssString}</style>. Cả hai chỉ cần thay giá trị hex/rgba bằng chuỗi var(--vy-…) theo bảng §5 — không đổi cấu trúc JSX, không đổi layout. Phương án B giữ nguyên backdrop-filter (header kính mờ, drawer) và ảnh không bao giờ bị âm bản — hai điểm yếu của phương án A.

2. Vì sao token phải mang prefix --vy-

Guide gốc đề xuất tên trần (--bg, --border, --muted…). Trong globals.css hiện tại, --border, --muted, --background… đã bị shadcn/Tailwind v4 chiếm (khối oklch + @theme inline, sinh utility border-border, bg-muted…). Định nghĩa đè sẽ phá form control admin. Ngoài ra còn bộ --nl-* legacy đang được hàng chục rule !important tham chiếu. Kết luận: bộ token mới dùng prefix --vy-, còn bộ --nl-* được trỏ về token vy (§6.2) — nhờ đó mọi rule legacy tự đổi theme mà không phải sửa từng dòng.

4. Nguyên tắc bất biến — cái gì KHÔNG đổi theo theme

Quy tắc ngón tay cái: màu nằm trên ẢNH thì giữ nguyên; màu nằm trên NỀN trang thì token hóa.

  • Gradient gold của logo wordmark, CTA "Đặt ngay/Đặt bàn", badge VIP, thẻ membership — nhận diện thương hiệu, giữ cả 2 theme; chữ trên đó luôn #241a0a.
  • Scrim phủ ảnh — mọi rgba(12,12,15,…) và gradient đen đáy ảnh quán/cast/hero: giữ nguyên để chữ trắng trên ảnh luôn đọc được (ảnh nightlife vốn tối).
  • Chữ/icon trắng nằm trên ảnh (tên quán trên hero, badge "ĐANG DIỄN RA"…): giữ #fff/#f0dda8.
  • Gradient fallback hero chi tiết quán (store-detail.helpers.ts, StoreDetailHeader.tsx, StoreDetailClient.tsx: linear-gradient(135deg,#18181c…#111114)) — đóng vai ảnh, trên nó có chữ trắng → giữ.
  • Nút play video trang chủ (dòng ~1156: nền kem rgba(243,240,234,.92), icon #0c0c0f) — cặp màu trên thumbnail video → giữ.
  • Khung ngoài mobile simulator (.nl-mobile-simulator-root #f6f5f2) — desk xám sáng bao quanh khung điện thoại, giữ.
  • rgba(0,0,0,…) bên trong gradient phủ ảnh — giữ; bóng đổ thì dùng token shadow khi tiện (không bắt buộc từng cái).

6.2 · globals.css — trỏ bộ --nl-* về token vy

Thay block :root { --nl-bg: #0c0c0f; … } (dòng ~154) bằng phần 2 của handoff/01-tokens.css. Hiệu ứng dây chuyền: html, body { background: var(--nl-bg) !important }, các rule remap [style*="background: #fff"]…, input/placeholder — tất cả tự ăn theme không cần sửa thêm.

6.3 · src/app/layout.tsx — script chống chớp (bắt buộc)

Thêm làm phần tử đầu tiên trong <head>, trước mọi <Script> — chạy trước paint nên chuyển trang không chớp màn:

<!-- Kết quả HTML phải có trong <head> sau khi render: -->
<script>
  (function () {
    try {
      var t = localStorage.getItem('vy-user-theme');
      if (!t && window.matchMedia('(prefers-color-scheme: light)').matches) t = 'light';
      if (t === 'light') document.documentElement.classList.add('vy-light');
    } catch (e) { /* private mode */ }
  })();
</script>

Trong layout.tsx viết bằng JSX qua dangerouslySetInnerHTML — copy nguyên văn từ handoff/02-layout-theme-init.tsx.txt, không gõ tay để tránh sai ký tự escape.

6.4 · Gỡ engine theme cũ public/theme.js (bắt buộc, tránh xung đột)

theme.js là engine của prototype cũ: đọc key nl_theme, dùng MutationObserver quét mọi [style] và remap hex sang bảng màu tím cũ, đồng thời set cứng nền #0f0d15 lên <html>/<body> — sẽ phá theme mới với user còn nl_theme='dark' trong máy. Nút của nó hiện đã bị ẩn bằng CSS.

  1. Xóa dòng <Script src="/theme.js" strategy="beforeInteractive" /> trong layout.tsx.
  2. Xóa file public/theme.js.
  3. (Tùy chọn) bỏ rule #nl-theme-btn { display: none !important; } trong globals.css. Không cần dọn key nl_theme của user cũ — engine gỡ rồi thì key tự chết.

Spec: tròn 39px (desktop) / 36px (mobile — vùng chạm 44px đã được rule .nl-site-actions min-height đảm bảo), viền --vy-border-gold-32, icon 17px màu --vy-gold, hover nền --vy-gold-soft-bg; đang tối hiện mặt trời, đang sáng hiện mặt trăng; có title + aria-label "Chuyển giao diện sáng/tối".
